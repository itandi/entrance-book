複数プロダクトの品質を支える「共通コンポーネント基盤」のフロントエンドエンジニアを募集します。

---

## 1. Mission / 私たちの役割
私たちは特定のプロダクト専任ではなく、全プロダクトを横断的に支える**「技術基盤（Platform）」チーム**です。

* **課題:** 各プロダクトチームは機能開発に追われており、最新のフロントエンド技術への追従や、コード品質の均質化が難しい現状があります。
* **解決策:** 私たちが「共通コンポーネント」「開発ポリシー」「デザインシステム」を提供し、組織全体の開発効率とUI品質を底上げします。

### チーム体制
* Frontend Engineer: 2名（★ここに3人目としてジョインしていただきます）
* UI Designer: 3名
* 各事業部のプロダクト開発チーム（Railsエンジニア中心）に対し、私たちが技術支援を行います。

---

## 2. Tech Stack / 技術スタック
「動けばいい」ではなく、**「型安全性」「再利用性」「メンテナンス性」**に徹底的にこだわります。

| Category | Technology |
| :--- | :--- |
| **Core** | React, TypeScript (Strict Mode), Next.js (Static Exports) |
| **Styling** | CSS Modules (移行中), Tailwind CSS |
| **Component** | **Storybook** (共通基盤の中核), Vite |
| **Quality** | **Biome** (Prettier/ESLintから移行中), Playwright |
| **Infra** | Vercel, AWS |

### 今、取り組んでいる技術的挑戦
* **MCP (Model Context Protocol) サーバーの自作**
    * ドキュメントを探す時間をゼロにするため、デザインシステムやテスト実装についてAIエージェントに質問すれば即答してくれる仕組みを構築中です。
* **Linter/Formatterの刷新**
    * 開発速度向上のため、Prettier/ESLintから **Biome** への完全移行を進めています。

---

## 3. Work Style / 働き方
「横断組織」だからこそ、密なコミュニケーションを重視しています。

* **勤務地:** 六本木グランドタワー
* **出社頻度:** 週3回（月・水・他1日）
    * **Why?** デザイナーとのUI調整や、各チームからの相談をクイックに解決するため、対面での「帯域の広い」コミュニケーションを大切にしています。
* **制度:** フルフレックス制度（コアタイムなし）
    * 自身の裁量で「昨日は残業したから今日は早上がり」といった調整が可能です。

---

## 4. FAQ / よくある質問

**Q. ユーザー向けの機能開発はできますか？**
* **A.** 基本的には行いません。
私たちは「ユーザー」ではなく「社内のエンジニアやデザイナー」を顧客とするチームです。プロダクトの裏側にある「仕組み」を作ることに喜びを感じる方に最適な環境です。

**Q. バックエンド（Ruby on Rails）は触りますか？**
* **A.** いいえ、触りません。
私たちはフロントエンドのエキスパートとして活動するため、Railsには触れず、TypeScript/Reactのエコシステムを深く追求します。
